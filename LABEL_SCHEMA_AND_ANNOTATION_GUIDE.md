# Схема разметки датасета для gost_formatter

Датасет предназначен для обучения NLP/ML-компонента системы автоматизированного оформления документов по требованиям ГОСТ.

## Объем

- Документов: 59
- Блоков всего: 34883
- Блоков в train-ready: 26859

## Уровни разметки

- `label_core` — основная метка для обучения ML-классификатора.
- `label_detailed` — уточняющая метка для анализа ошибок и rule-based слоя.

## Основные классы

| Метка | Назначение |
|---|---|
| `body_text` | основной текст |
| `title_section` | заголовок раздела |
| `title_subsection` | заголовок подраздела |
| `title_subsubsection` | заголовок пункта/подпункта |
| `list_item` | элемент списка |
| `figure_caption` | подпись к рисунку |
| `table_caption` | подпись к таблице |
| `bibliography_title` | заголовок списка источников |
| `bibliography_item` | элемент списка источников |
| `appendix_title` | заголовок приложения |
| `toc_title` | заголовок содержания |
| `toc_item` | строка содержания |
| `table_cell` | текст внутри таблицы |
| `formula` | формульный блок |
| `candidate_heading` | кандидат в заголовок, требует проверки |
| `ignore` | пустой или технический блок |

## Что проверить вручную в первую очередь

1. `candidate_heading`.
2. `bibliography_item`.
3. `formula`.
4. `appendix_title`.
5. `table_caption` и `figure_caption`.

Разметка является weak-label: она полезна для стартового обучения, но финальные метрики нужно считать после ручной валидации спорных блоков.
