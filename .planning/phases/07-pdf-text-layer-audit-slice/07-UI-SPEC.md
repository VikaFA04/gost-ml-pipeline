# Phase 07 — UI Design Contract

> Visual and interaction contract for the PDF text-layer audit slice. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Phase 7 is a **small additive UI surface** on top of the Phase 6 Streamlit redesign — it adds PDF input acceptance and three audit-only signals. Every design token (spacing, typography, color, status chips, registry safety) is **inherited verbatim from `06-UI-SPEC.md`**. This document declares ONLY what changes for Phase 7.

Locked decisions in `07-CONTEXT.md` (D-01..D-04 + D-PDF-SCOPE) are normative and pre-populated below. Sections marked **Claude's discretion** are decided here and become normative for `gsd-planner` and `gsd-executor`.

Every dimension traces to REQ-pdf-text-only and the four Phase 7 success criteria (SC-1..SC-4 in `ROADMAP.md`).

---

## Scope Reality Check

Phase 7 is **NOT a redesign**. There is no new page, no new component library, no new theming. The UI delta is exactly four edits to `app.py` plus one README section:

1. **`SUPPORTED_UPLOAD_TYPES`** — extend `["docx"]` → `["docx", "pdf"]` (uploader accepts `.pdf` alongside `.docx`).
2. **Preflight Russian message** — register a new branch in `preflight_translate_error` for the PDF-no-text-layer rejection path.
3. **Audit-only badge** in `render_report` header when input is PDF.
4. **Sidebar note** under uploader stating PDF is audit-only.
5. **README §Limits** paragraph stating text-layer + read-only + no-OCR boundaries (SC-4).

Everything else (Phase 6 status chips, spacing, typography, color palette, registry safety) **stays unchanged**. The executor MUST NOT introduce new tokens, palettes, or component libraries.

- **«Применённые исправления»** — empty/«—» (always per SC-2 «`applied_fixes` is always empty»).

Removal of the dead branch is mandatory; leaving it would be dead code (CLAUDE.md «Удаляй orphans, появившиеся из-за твоих изменений»).

**RunLog stage labels for the PDF preflight rejection path** (Claude's discretion per `07-CONTEXT.md`):

| Trigger | `stage` | `status` | `error_class` |
|---------|---------|----------|---------------|
| PDF text-layer < 50% threshold | `document-read` | `error` | `PdfNoTextLayer` (or equivalent typed exception name — executor picks the exact class name) |
| PDF accepted, classifier skipped (D-02 no-op path) | `classification` | `ok` | null (no-op succeeds — preserves the 4-stage shape) |
| PDF accepted, rule-apply skipped (every block routes to `review` with no rule engine call) | `rule-apply` | `ok` | null (no-op succeeds — preserves the 4-stage shape) |
| PDF accepted, save phase emits report CSV + summary JSON only (no DOCX write) | `save` | `ok` | null |

**Rationale for keeping the 4-stage shape on PDF input:** the run-log JSON contract from `06-UI-SPEC.md` (REQ-pipeline-logging + D-04 PII boundary) declares `stage ∈ {document-read, classification, rule-apply, save}` as a literal enum. Adding a `pdf-extract` stage would force every Phase 6 consumer (run-log download UI, future log-aggregation tooling) to widen the enum. Keeping the existing 4-stage shape — and recording PDF's no-op classification/rule-apply as `status="ok"` rather than introducing new stages — preserves backward compatibility and keeps the run-log machine-readable contract stable.

### README §Limits update (D-04 §3 + SC-4)

A new paragraph added under an existing or new `## Limits` heading in `README.md`. Content contract (must include all three boundaries per SC-4):

```
## Limits

PDF input is supported in audit-only mode: the document must contain an extractable text layer (≥ 50% of pages must return non-empty text). Scanned PDFs and page-image PDFs are rejected at preflight — OCR is not supported. No corrected PDF is produced; `applied_fixes` is always empty for PDF blocks. DOCX input remains the only path that produces a corrected document.
```

**Language: English** (Claude's discretion per `07-CONTEXT.md`). Rationale:
- `README.md` is developer-facing (CI/contributor audience).
- `CONTRIBUTING.md` (Phase 4) is already English.
- In-app Russian copy is enforced by the locked sidebar note + audit-only badge + preflight rejection message; users who only see the UI never need to read the README.
- Writing both Russian and English would duplicate maintenance burden and violate CLAUDE.md «Пиши минимум кода/документации».

The README paragraph must include the substrings **«audit-only»**, **«text layer»**, **«no OCR»** (or «OCR is not supported»), and **«no corrected PDF»** (or equivalent) — these are the three boundaries SC-4 mandates.
