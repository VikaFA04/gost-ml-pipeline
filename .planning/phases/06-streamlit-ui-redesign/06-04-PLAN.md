<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Methodical-file upload → save_uploaded_bytes → build_methodical_profile | User-supplied PDF/DOCX/TXT/MD bytes parsed by external libraries; size and content not under app control |
| Modal save → CUSTOM_PROFILES_DIR | Writes a JSON file to results/generated_profiles/ — path fixed, profile_id derived from the parsed content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-6-03 | Tampering (MEDIUM) | Modal allows silent profile overwrite | mitigate | (1) Apply button is disabled until BOTH the overwrite checkbox is ticked AND modal_reason_is_valid(reason) returns True. modal_reason_is_valid enforces strip-len ≥ 8 (T-05-01 client-side). (2) draft['extraction_meta']['override_reason'] is set to reason.strip() before save — mirrors CLI cmd_extract_methodical_profile lines 367-375. (3) Acceptance criterion `grep -q "override_reason" app.py` proves the field write. (4) D-004 «no silent rewrites» enforced by the disabled-button gate. |
| T-6-04 | Denial of service (LOW) | Modal accepts large PDF | mitigate | save_uploaded_bytes writes via tempfile.NamedTemporaryFile — bounded by OS temp space + Streamlit's default 200MB maxUploadSize. build_methodical_profile is synchronous and may run > 30s on very large PDFs (Risk 2 in 06-RESEARCH.md §9); acceptable for MVP. A 50MB soft warn is a UI-SPEC discretion item, not implemented in 06-04 — deferred. |
| T-6-07 | Path traversal via uploaded filename | save_uploaded_bytes / CUSTOM_PROFILES_DIR write | accept | save_uploaded_bytes uses tempfile.NamedTemporaryFile with `suffix=Path(uploaded.name).suffix` only — the suffix is extracted but the path is OS-controlled. CUSTOM_PROFILES_DIR is fixed at `results/generated_profiles/`; the file name is `{profile_id}.json` where profile_id is derived from the parsed methodical content, NOT from the upload filename. Path traversal in profile_id is theoretically possible via crafted PDF metadata; mitigation is upstream (methodical_extractor sanitizes profile_id) — Phase 5 T-04-02 covered the CLI path; UI is no worse. |
| T-6-08 | XSS via diff rendering | st.code(joined_lines, language=None) | mitigate | st.code renders monospace-pre text, not HTML. compute_profile_diff returns only ASCII + Cyrillic + the U+2192 arrow + path separators — no HTML interpretation occurs. Verified by Streamlit primitive choice. |
</threat_model>
